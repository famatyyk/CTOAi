# Daily 24/7 Loop

Backlog aktywny: `workflows/backlog-sprint-001.yaml`
Runbook wykonawczy: `workflows/runbook-24x7.yaml`

1. Co 15 min: sprawdz kolejke zadan i health agentow.
2. Co 1h: mini-cykl Plan -> Build -> Test -> Gate.
3. Co 24h: raport dzienny i repriorytetyzacja.
4. Co 20 dni: release aplikacji.
5. Zawsze: publikacja tylko po recznej akceptacji ownera.
